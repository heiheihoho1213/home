MontoApp（六） - js 沙箱：自执行 Script 与 快照隔离

本节继续，介绍微前端中 js 隔离的方案。前一讲的 iframe 隔离，本质上是利用了 V8 引擎的隔离机制，借助 iframe 实现的；本节说的快照隔离，本质上还是把子应用执行在主应用上下文中，但是对主应用的 winodw 做快照，在子应用卸载的时候恢复快照即可。

1. 快照隔离沙箱原理

4. 实现沙箱

沙箱的实现步骤可以参见 iframe 沙箱的流程，唯一不同的是 Sandbox 类的实现。

我们实现沙箱类构造函数：

1// 类主要成员变量 2// 微应用 JS 运行之前的主应用 window 快照 3mainWindow = {}; 4// 微应用 JS 运行之后的 window 对象（用于理解） 5microWindow = {}; 6// 微应用失活后和主应用的 window 快照存在差异的属性集合 7diffPropsMap = {}; 8 9constructor(options) { 10 this.options = options; 11 // 重新包装需要执行的微应用 JS 脚本 12 this.wrapScript = this.createWrapScript(); 13} 14 15createWrapScript() { 16 // 微应用的代码运行在立即执行的匿名函数中，隔离作用域 17 return `;(function(window){ 18 ${this.options.scriptText} 19 })(window)`; 20}

同样是闭包执行脚本，不同的是不再使用 window 的代理了，这里的 window 是主应用 window 的快照，下面会介绍实现。

激活

沙箱创建完成后，就是激活和卸载功能了，先看激活：

1active() { 2 // 记录微应用 JS 运行之前的主应用 window 快照 3 this.recordMainWindow(); 4 // 恢复微应用需要的 window 对象 5 this.recoverMicroWindow(); 6 if (this.exec) { 7 return; 8 } 9 this.exec = true; 10 // 执行微应用（注意微应用的 JS 代码只需要被执行一次） 11 this.execWrapScript(); 12}

其中 recordMainWindow 用于记录挂载之前主应用的 window 状态：

1recordMainWindow() { 2 for (const prop in window) { 3 if (window.hasOwnProperty(prop)) { 4 this.mainWindow[prop] = window[prop]; 5 } 6 } 7}

recoverMicroWindow 是恢复上一次挂载子应用时的 window 状态：

1recoverMicroWindow() { 2 // 如果微应用和主应用的 window 对象存在属性差异 3 // 上一次微应用 window = 主应用 window + 差异属性（在微应用失活前会记录运行过程中涉及到更改的 window 属性值，再次运行之前需要恢复修改的属性值） 4 Object.keys(this.diffPropsMap).forEach((p) => { 5 // 更改 JS 运行之前的微应用 window 对象，注意微应用本质上共享了主应用的 window 对象，因此一个时刻只能运行一个微应用 6 window[p] = this.diffPropsMap[p]; 7 }); 8 9 this.microWindow = window; 10}

然后就是执行 脚本 了：

1execWrapScript() { 2 // 在全局作用域内执行微应用代码 3 (0, eval)(this.wrapScript); 4}

(0, eval) 是一个逗号表达式，其返回最右边的值，等价于 eval，之所以这么写，是因为逗号表达式之后返回的是一个对 eval 的间接调用，而间接调用计算出来的是一个值，而不是引用，相当于把这个函数单独声明出来再执行。

上面的代码 (0, eval)(this.wrapScript)，相当于改变了 eval 执行对象为 wrapScript，改变了内部执行的 this 指针。

失效

我们再来看看解除激活状态的实现。

1inactive() { 2 // 清空上一次记录的属性差异 3 this.diffPropsMap = {}; 4 // 记录微应用运行后和主应用 Window 快照存在的差异属性 5 this.recordDiffPropsMap(); 6 console.log( 7 `${this.options.appId} diffPropsMap: `, 8 this.diffPropsMap 9 ); 10}

其中 recordDiffPropsMap：

1recordDiffPropsMap() { 2 // 这里的 microWindow 是微应用失活之前的 window（在微应用执行期间修改过 window 属性的 window） 3 for (const prop in this.microWindow) { 4 // 如果微应用运行期间存在和主应用快照不一样的属性值 5 if ( 6 window.hasOwnProperty(prop) && 7 this.microWindow[prop] !== this.mainWindow[prop] 8 ) { 9 // 记录微应用运行期间修改或者新增的差异属性（下一次运行微应用之前可用于恢复微应用这一次运行的 window 属性） 10 this.diffPropsMap[prop] = this.microWindow[prop]; 11 // 恢复主应用的 window 属性值 12 window[prop] = this.mainWindow[prop]; 13 } 14 } 15}

大致流程就是记录一下与原来 window 的差异，然后恢复原来记录的主应用的 window。

5. 小结

本隔离方式实际是动态 script 的变种，没有创建 script 标签，而是直接在 window 快照中执行微应用的 js，缺点很明显：

  • 不能同时加载多个子应用
  • 无法处理主子应用同时操作 window 产生冲突的问题
