Gitlab安装与仓库管理

在团队代码私有仓库中，互联网公司中gitlab应该是首选的，不仅仅是个仓库，其支持私有化部署和具备独有的CI/CD，优势明显。gitlab 有免费的社区版（CE）也有收费的企业版（EE），国内还有极狐版（JH）。本文以 CE 为例讲解。

本文介绍如何在服务器（以centOS7为例）上安装gitlab，并实现web端账号配置与ssh配置仓库。

国内可以使用 gitlab 在国内的代理 极狐 来安装，很多设置都已经做好了，也不存在网络问题，可以省去很多繁重的配置工作。本文还是以清华的镜像源为例。

1. 安装前检查

在安装前，可以检查一下是否安装有旧版本，卸载掉他！

按顺序执行如下指令：

1## 停止gitlab 2gitlab-ctl stop 3 4## 卸载 (请确定版本`ce/ee/jh`等。若找不到包说明上次没有安装成功) 5rpm -e gitlab-ce 6 7## 检查一下进程 8ps aux | grep gitlab

1## 清除掉 service 的进程 2kill -9 9956 3 4## 删除所有包含gitlab文件 （递归查找有些慢，可以不做） 5find / -name gitlab | xargs rm -rf

2. 安装

  • 操作系统：CentOS7
  • gitlab：GitLab-ce 清华源镜像 （gitlab-ce-11.5.3-ce.0.el7.x86_64.rpm）

前置安装

在系统防火墙中打开 HTTP、HTTPS 和 SSH 访问。如果打算仅从本地网络访问极狐GitLab，则可以跳过它。

1## 全部粘贴一起丢到终端里 2sudo yum install -y curl policycoreutils-python openssh-server perl 3sudo systemctl enable sshd 4sudo systemctl start sshd 5 6sudo firewall-cmd --permanent --add-service=http 7sudo firewall-cmd --permanent --add-service=https 8sudo systemctl reload firewalld

ssh配置：

1sudo yum install curl policycoreutils openssh-server openssh-clients

配置邮件通知（可选）

1sudo yum install postfix 2sudo systemctl enable postfix 3sudo systemctl start postfix

修改邮件配置：vi /etc/postfix/main.cf

1inet_interfaces = all            //之前应该是localhost

启动 postfix：

1service postfix start 2systemctl status postfix.service 3sudo systemctl enable postfix

由于国内 wget速度较慢，可以使用 mwget：(这一堆丢到终端里)

1wget http://jaist.dl.sourceforge.net/project/kmphpfm/mwget/0.1/mwget_0.1.0.orig.tar.bz2 2yum install bzip2 gcc-c++ openssl-devel intltool -y 3bzip2 -d mwget_0.1.0.orig.tar.bz2 4tar -xvf mwget_0.1.0.orig.tar 5cd mwget_0.1.0.orig 6./configure 7make 8make install　

下载与安装 gitlab

下载镜像源：

1mwget https://mirrors.tuna.tsinghua.edu.cn/gitlab-ce/yum/el7/gitlab-ce-11.5.3-ce.0.el7.x86_64.rpm

安装：

1rpm -i gitlab-ce-11.5.3-ce.0.el7.x86_64.rpm

3. 安装后配置

重置root账户

编辑 /etc/gitlab/gitlab.rb（如果不存在则创建）并设置：

1gitlab_rails['initial_root_password'] = 'passport'

同目录下的 initial_root_password 也修改一下，如果有的话。当然也可以重置密码：

1## 1. 获取管理员权限 2sudo su 3## 2. 进入控制台 4gitlab-rails console production 5## 3. 查找root用户 6user = User.where(id: 1).first 7## 4. 设置 8user.password = 'passport' #设置新的密码 9user.password_confirmation = 'passport' #确认密码 10## 5. 保存 11user.save! 12## 当发现=> true时表示重置成功 13 14## 6. 退出 15ctrl+d or exit

配置web URL

编辑 /etc/gitlab/gitlab.rb

1## 配置本机ip可访问的地址 2external_url "http://127.0.0.1:10086"

最后生效配置：

1gitlab-ctl reconfigure

查看状态

1gitlab-ctl status

5. 客户端访问

设置 CentOS 外部可访问：

执行指令：

1iptables -F 2iptables -P INPUT ACCEPT（设置默认允许规则）

配置防火墙，允许10086端口访问：

1## /etc/sysconfig/iptables-config 2 3## 添加 4-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp --dport 10086 -j ACCEPT

开放端口：

1firewall-cmd --zone=public --add-port=10086/tcp --permanent
