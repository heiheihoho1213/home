React Router v6 官方文档翻译 （七） ---- 内置API

本期讲一下 React Router v6 的utils库中的内置方法。本文只在掘金发布。

专栏说明：对于react-router v6版本，有同事反映缺少相对完整的中文文档，查看不方便。为了便于使用，作者对官网文档进行针对性翻译，便于v6版本更广泛的被使用。

1. createRoutesFromChildren

类型定义：

1declare function createRoutesFromChildren( 2 children: React.ReactNode 3): RouteObject[]; 4 5// 返回值 6interface RouteObject { 7 caseSensitive?: boolean; 8 children?: RouteObject[]; 9 element?: React.ReactNode; 10 index?: boolean; 11 path?: string; 12}

createRoutesFromChildren 通常在 <Route> 子元素中使用，用于生成子 route 的配置项。可以把 <Route> 类型的 react element 对象，变成了普通的 route 对象结构。其内部通过 React.Children.forEach 把 Route 组件给结构化，并且内部调用递归，深度递归 children 结构。

示例：

输入

1<Routes> 2 <Route element={<Home />} path="/home" /> 3 <Route element={<List/>} path="/list" /> 4 <Route element={<Layout/>} path="/children" > 5 <Route element={<Child1/>} path="/children/child1" /> 6 <Route element={<Child2/>} path="/children/child2" /> 7 </Route> 8</Routes>

2. createSearchParams

类型定义：

1declare function createSearchParams( 2 init?: URLSearchParamsInit 3): URLSearchParams;

createSearchParams 通常与 useSearchParams 配对使用，用于 search 传参时构建参数。

示例：

1// 传参方式 1 2navigate({ 3 pathname: "/class", 4 search: `?id=${id}&grade=${grade}` 5}) 6 7// 传参方式 2 8const params = { id: '1', grade: '2' }; 9navigate({ 10 pathname: "/class", 11 search: `?${createSearchParams(params)}` 12})

3. generatePath

类型定义：

1declare function generatePath( 2 path: string, 3 params?: Params 4): string;

动态生成可跳转的 path， 可自动匹配并填充动态参数。

示例：

1generatePath("/users/:id", { id: "42" }); // "/users/42" 2generatePath("/files/:type/*", { 3 type: "img", 4 "*": "cat.jpg", 5}); // "/files/img/cat.jpg"

5. matchPath

类型定义：

1declare function matchPath< 2 ParamKey extends string = string 3>( 4 pattern: PathPattern | string, 5 pathname: string 6): PathMatch<ParamKey> | null; 7 8interface PathMatch<ParamKey extends string = string> { 9 params: Params<ParamKey>; 10 pathname: string; 11 pattern: PathPattern; 12} 13 14interface PathPattern { 15 path: string; 16 caseSensitive?: boolean; 17 end?: boolean; 18}

比较常用的函数。是hook [useMatch](https://reactrouter.com/en/main/hooks/use-match) 的补充，用于非函数式组件中。

示例：

6. matchRoutes

类型定义：

1declare function matchRoutes( 2 routes: RouteObject[], 3 location: Partial<Location> | string, 4 basename?: string 5): RouteMatch[] | null; 6 7interface RouteMatch<ParamKey extends string = string> { 8 params: Params<ParamKey>; 9 pathname: string; 10 route: RouteObject; 11}

这个是v6版匹配路由的核心算法，官方暴露出来了，供开发者自定义使用。

示例：

1const routes = [ 2 { 3 path: '/', 4 component: <Home /> 5 }, 6 { 7 path: '/list', 8 component: <List /> 9 }, 10 { 11 path: '/children', 12 component: <Layout />, 13 children: [ 14 { 15 path: 'child1', 16 component: <Child1 /> 17 }, 18 { 19 path: 'child2/:id', 20 component: <Child2 /> 21 }, 22 ] 23 } 24] 25... 26 27<BrowserRouter> 28 <Routes> 29 {routes.map((route, index) => <Route key={index} element={route.component} path={route.path} > 30 {route.children && <> 31 {route.children.map((cRoute, cIndex) => <Route key={cIndex} element={cRoute.component} path={cRoute.path} />)} 32 </>} 33 </Route> )} 34 </Routes> 35</BrowserRouter> 36 37matchRoutes(routes, '/children/child1')

可以看到， matchRoutes 将所有的匹配路径上的路由都按照顺序返回啦。可以通过 route.path 来获取原始路由表中的 path。

7. renderMatches

类型定义：

1declare function renderMatches( 2 matches: RouteMatch[] | null 3): React.ReactElement | null;

它的作用是把 matchRoutes 匹配到的 component 渲染出来。说的容易，网上的使用案例少之又少，官方文档又太过敷衍，只能去读源码了：

1// react-router/packages/lib/hooks.tsx 2export function _renderMatches( 3 matches: RouteMatch[] | null, 4 parentMatches: RouteMatch[] = [] 5): React.ReactElement | null { 6 if (matches == null) return null; 7 8 return matches.reduceRight((outlet, match, index) => { 9 return ( 10 <RouteContext.Provider 11 children={ 12 match.route.element !== undefined ? match.route.element : outlet 13 } 14 value={{ 15 outlet, 16 matches: parentMatches.concat(matches.slice(0, index + 1)), 17 }} 18 /> 19 ); 20 }, null as React.ReactElement | null); 21}

可以看到，它是将匹配到的路由数组从右向左摘取，统一放到一个 context 里了。而且，从源码里看到，useRoutes 的返回值就是一个 _renderMatches，我们可以参照源码的调用方式进行使用。

考虑路由：

1const routes = [ 2 { 3 path: '/', 4 component: <Home /> 5 }, 6 { 7 path: '/list', 8 component: <List /> 9 }, 10 { 11 path: '/children', 12 component: <Layout />, 13 children: [ 14 { 15 path: 'child1', 16 component: <Child1 /> 17 }, 18 { 19 path: 'child2/:id', 20 component: <Child2 /> 21 }, 22 ] 23 }, 24 ]

使用方式：

1renderMatches(matchRoutes(routes, '/children/child2/1'))

matchRoutes 返回的值是：

与 _renderMatches 接受的参数一致。

在 matches.reduceRight 中，会将 match.route.element 通过 context 的 children 的形式挂载。不知道是不是开发者的笔误，children 里使用了 match.route.element，而上图中匹配的数组明明是 route.component，导致直接使用渲染不出来任何数据，我改动一下这个函数就有数据了：

1const Context = createContext({}); 2 3const matches = matchRoutes(routes, '/children/child2/1'); 4 5return matches && matches.reduceRight((outlet, match, index) => { 6 return ( 7 <Context.Provider 8 children={ 9 match.route.component !== undefined ? match.route.component : outlet 10 } 11 value={{ 12 outlet, 13 }} 14 /> 15 ); 16}, null as React.ReactElement | null);

但是他传的这个 outlet 参数，并不能被自动识别为 <Outlet /> 标签，所以只能渲染出来 Layout， 子组件仍然渲染不出来。

译者自己写了个Demo，来复现一下他这个算法：

1// 主 2<Context.Provider 3 value={{ 4 outlet: <Context.Provider> 5 <Child2 /> 6 </Context.Provider> 7 }} 8> 9 <Layout /> 10</Context.Provider> 11 12// Layout 13function Layout () { 14 return <Context.Consumer> 15 {values => { 16 return <> 17 'Layout 18 {values.outlet ? values.outlet : <Outlet />} 19 </> 20 }} 21 </Context.Consumer> 22}

总结：renderMatches 不要用，坑人。嵌套 Outlet 的思想很好，但终究是给内部使用的内部 API，业务上不要用。

8. resolvePath

类型定义：

1declare function resolvePath( 2 to: To, 3 fromPathname?: string 4): Path; 5 6type To = string | Partial<Path>; 7 8interface Path { 9 pathname: string; 10 search: string; 11 hash: string; 12}

通过给定的相对路由 to 来获取一个能够 navigate 的绝对路径 path。是hook useResolvedPath 的实现方法。

1resolvePath('children/child2/:id') // /children/child2/:id
