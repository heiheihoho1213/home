CSS 变量

这个方案是目前主流推荐的方案。在介绍之前，先讲一下浏览器的几个新特性。

color-scheme 方案

color-scheme 是浏览器普遍支持的配置主题的属性。他表示调用操作系统默认的主题方案覆盖当前元素样式：

color-scheme：CSS 变量主题切换案例

自定义方案

如果要自定义变量，可以这样定义自己的样式变量：

1// theme.css 2 3// 夜间模式 4html[data-theme='dark'] { 5 --text-color: #fff; 6 --brand-primary: #1668dc; 7} 8 9// 普通模式 10:root { 11 --text-color: #333; 12 --brand-primary: #1677ff; 13}

然后在使用时：

1// 入口文件 2import './theme.css'; 3 4// 组件样式 5a { 6 color: var(--brand-primary); 7}

在切换主题时：

1document.documentElement.dataset.theme = 要设置的theme

你可以维护一个 useTheme 来动态获取当前的主题。

这里有一个成熟的案例：daisyUI

优点：

  • 原生支持，是现在主流方案
  • 轻量级，部署方便，可定制化程度高
  • 不存在优先级冲突问题
  • 可实现热替换和更改

缺点：

  • 老的浏览器存在兼容性问题
