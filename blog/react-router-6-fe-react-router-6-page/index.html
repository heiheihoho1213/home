React Router v6 官方文档翻译 （六） ---- Hooks

专栏说明：对于react-router v6版本，有同事反应缺少相对完整的中文文档，查看不方便。为了便于使用，作者对官网文档进行针对性翻译，便于v6版本更广泛的被使用。

作者说明：本期的hooks仅仅在函数式组件里生效，若想要在类式组件里使用相应功能，可参见本专栏第一篇文章，自己写一个 withRouter 函数。

1. useHref

类型定义：

1declare function useHref(to: To): string;

useHref 返回一个完整的路由URL，指向参数 to 定位的地址, 这个地址可以脱离 React Router环境。使用示例：

1// 深层路由中 2useHref({ to: 'dashboard' }) // /home/test/dashboard

2. useInRouterContext

类型定义：

1declare function useInRouterContext(): boolean;

当组件使用了<Router>渲染时，该钩子返回true，否则返回false。这对于一些需要知道组件是否在 React Router 上下文中的第三方扩展非常有用。

3. useLinkClickHandler

类型定义：

1declare function useLinkClickHandler< 2 E extends Element = HTMLAnchorElement 3>( 4 to: To, 5 options?: { 6 target?: React.HTMLAttributeAnchorTarget; 7 replace?: boolean; 8 state?: any; 9 } 10): (event: React.MouseEvent<E, MouseEvent>) => void;

其返回一个能够实现<Link>标签功能的替代性的函数，用于在自定义跳转时使用，参数同Link标签：

1import { 2 useHref, 3 useLinkClickHandler, 4} from "react-router-dom"; 5 6const StyledLink = styled("a", { color: "fuchsia" }); 7 8const Link = React.forwardRef( 9 ( 10 { 11 onClick, 12 replace = false, 13 state, 14 target, 15 to, 16 ...rest 17 }, 18 ref 19 ) => { 20 let href = useHref(to); 21 // 调用handleClick，相当于点击了一个<Link>标签 22 let handleClick = useLinkClickHandler(to, { 23 replace, 24 state, 25 target, 26 }); 27 28 return ( 29 <StyledLink 30 {...rest} 31 href={href} 32 onClick={(event) => { 33 onClick?.(event); 34 if (!event.defaultPrevented) { 35 handleClick(event); 36 } 37 }} 38 ref={ref} 39 target={target} 40 /> 41 ); 42 } 43);

4. useLinkPressHandler

类型定义：

1declare function useLinkPressHandler( 2 to: To, 3 options?: { 4 replace?: boolean; 5 state?: any; 6 } 7): (event: GestureResponderEvent) => void;

功能同 useLinkClickHandler， 只不过是在移动端使用的（常用于 react-native），使用示例：

1import { TouchableHighlight } from "react-native"; 2import { useLinkPressHandler } from "react-router-native"; 3 4function Link({ 5 onPress, 6 replace = false, 7 state, 8 to, 9 ...rest 10}) { 11 let handlePress = useLinkPressHandler(to, { 12 replace, 13 state, 14 }); 15 16 return ( 17 <TouchableHighlight 18 {...rest} 19 onPress={(event) => { 20 onPress?.(event); 21 if (!event.defaultPrevented) { 22 handlePress(event); 23 } 24 }} 25 /> 26 ); 27}

5. useLocation

类型定义：

1declare function useLocation(): Location; 2 3interface Location extends Path { 4 state: unknown; 5 key: Key; 6}

这个就比较常用了。他返回当前路由状态下的 location 信息, 可用于路由传参。使用方式：

1import { useLocation } from 'react-router-dom'; 2 3function App() { 4 let location = useLocation(); // hash, key, pathname, search, state 5}

7. useNavigate

类型定义：

1declare function useNavigate(): NavigateFunction; 2 3interface NavigateFunction { 4 ( 5 to: To, 6 options?: { replace?: boolean; state?: any } 7 ): void; 8 (delta: number): void; 9}

他返回一个可以用于路由导航的函数。有一个参数 replace，如果传为 true，路由会在 history stack 中替换当前的 current， 而不是新 push 一个状态。使用示例：

1import { useNavigate } from "react-router-dom"; 2 3// 表单提交后跳转页面 4function SignupForm() { 5 let navigate = useNavigate(); 6 7 async function handleSubmit(event) { 8 event.preventDefault(); 9 await submitForm(event.target); 10 navigate("../success", { replace: true }); 11 } 12 13 return <form onSubmit={handleSubmit}>{/* ... */}</form>; 14}

当然也可以不传路由的 path，直接传数字：

1navigate(-1)

表示浏览器回退或者前进。

8. useNavigationType

类型定义：

1declare function useNavigationType(): NavigationType; 2 3type NavigationType = "POP" | "PUSH" | "REPLACE";

返回路由跳转的状态，表示当前页面是怎么样被用户跳转过来的，可选参数：pop, push, replace

9. useOutlet

类型定义：

1declare function useOutlet(): React.ReactElement | null;

返回嵌套路由中，被 <Outlet> 占位的子元素的路由对象。通常在子路由内部使用。示例：

1function Dashboard() { 2 // 打印 outlet 3 console.log(useOutlet()) 4 return ( 5 <div> 6 <h1>Dashboard</h1> 7 <Outlet /> 8 </div> 9 ); 10} 11 12function DashboardMessages() { 13 return 'DashboardMessages' 14} 15 16function DashboardTasks() { 17 return 'DashboardTasks' 18} 19 20function App() { 21 return ( 22 <Routes> 23 <Route path="/" element={<Dashboard />}> 24 {/* 子路由 */} 25 <Route 26 path="messages" 27 element={<DashboardMessages />} 28 /> 29 <Route path="tasks" element={<DashboardTasks />} /> 30 </Route> 31 </Routes> 32 ); 33}

10. useOutletContext

类型定义：

1declare function useOutletContext< 2 Context = unknown 3>(): Context;

用于父子路由共享状态。示例：

1// 父 2function Parent() { 3 const [count, setCount] = React.useState(0); 4 return <Outlet context={[count, setCount]} />; 5} 6 7// 子 8import { useOutletContext } from "react-router-dom"; 9 10function Child() { 11 const [count, setCount] = useOutletContext(); 12 const increment = () => setCount((c) => c + 1); 13 return <button onClick={increment}>{count}</button>; 14}

如果是在ts环境下，使用方式又有所不同。推荐自定义一个 context， 这样方便类型定义：

1// 父 2import * as React from "react"; 3import type { User } from "./types"; 4import { Outlet, useOutletContext } from "react-router-dom"; 5 6type ContextType = { user: User | null }; 7 8export default function Dashboard() { 9 const [user, setUser] = React.useState<User | null>(null); 10 11 return ( 12 <div> 13 <h1>Dashboard</h1> 14 <Outlet context={{ user }} /> 15 </div> 16 ); 17} 18 19// 自定义 hook 20export function useUser() { 21 return useOutletContext<ContextType>(); 22} 23 24// 子 25import { useUser } from "../dashboard"; 26 27export default function DashboardMessages() { 28 const { user } = useUser(); 29 return ( 30 <div> 31 <h2>Messages</h2> 32 <p>Hello, {user.name}!</p> 33 </div> 34 ); 35}

11. useParams

类型定义：

1declare function useParams< 2 K extends string = string 3>(): Readonly<Params<K>>;

返回路由传参的键值对参数（不是问号传参，问号的在location的search里）。子路由也会继承父路由的所有参数。使用示例：

1import * as React from 'react'; 2import { Routes, Route, useParams } from 'react-router-dom'; 3 4function ProfilePage() { 5 // Get the userId param from the URL. 6 let { userId } = useParams(); 7 // ... 8} 9 10function App() { 11 return ( 12 <Routes> 13 <Route path="users"> 14 <Route path=":userId" element={<ProfilePage />} /> 15 <Route path="me" element={...} /> 16 </Route> 17 </Routes> 18 ); 19}

12. useResolvedPath

类型定义：

1declare function useResolvedPath(to: To): Path;

返回传入path的全路径。可解析出path，hash，search（location对象）

13. useRoutes

类型定义：

1declare function useRoutes( 2 routes: RouteObject[], 3 location?: Partial<Location> | string; 4): React.ReactElement | null;

是标签 <Routes> 的函数式形式。使用时也不需要额外的JSX。示例：

1import * as React from "react"; 2import { useRoutes } from "react-router-dom"; 3 4function App() { 5 let element = useRoutes([ 6 { 7 path: "/", 8 element: <Dashboard />, 9 children: [ 10 { 11 path: "messages", 12 element: <DashboardMessages />, 13 }, 14 { path: "tasks", element: <DashboardTasks /> }, 15 ], 16 }, 17 { path: "team", element: <AboutPage /> }, 18 ]); 19 20 return element; 21}

他的返回值要么是匹配到的React元素，要么是null。

14. useSearchParams

类型定义：

1declare function useSearchParams( 2 defaultInit?: URLSearchParamsInit 3): [URLSearchParams, SetURLSearchParams]; 4 5type ParamKeyValuePair = [string, string]; 6 7type URLSearchParamsInit = 8 | string 9 | ParamKeyValuePair[] 10 | Record<string, string | string[]> 11 | URLSearchParams; 12 13type SetURLSearchParams = ( 14 nextInit?: URLSearchParamsInit, 15 navigateOpts?: : { replace?: boolean; state?: any } 16) => void;

解析当前路径，返回location中的search；并可重新设置 search 并触发跳转。示例：

1import * as React from "react"; 2import { useSearchParams } from "react-router-dom"; 3 4function App() { 5 let [searchParams, setSearchParams] = useSearchParams(); 6 7 function handleSubmit(event) { 8 event.preventDefault(); 9 let params = serializeFormQuery(event.target); 10 setSearchParams(params); 11 } 12 13 return ( 14 <div> 15 <form onSubmit={handleSubmit}>{/* ... */}</form> 16 </div> 17 ); 18}

其中， setSearchParams 类似于 navigate，但是仅仅传递 search 参数。

15. useSearchParams (React Native)

类型定义：

1declare function useSearchParams( 2 defaultInit?: URLSearchParamsInit 3): [URLSearchParams, SetURLSearchParams]; 4 5type ParamKeyValuePair = [string, string]; 6 7type URLSearchParamsInit = 8 | string 9 | ParamKeyValuePair[] 10 | Record<string, string | string[]> 11 | URLSearchParams; 12 13type SetURLSearchParams = ( 14 nextInit?: URLSearchParamsInit, 15 navigateOpts?: : NavigateOptions 16) => void; 17 18interface NavigateOptions { 19 replace?: boolean; 20 state?: any; 21}

使用方式同上。示例：

1import * as React from "react"; 2import { View, SearchForm, TextInput } from "react-native"; 3import { useSearchParams } from "react-router-native"; 4 5function App() { 6 let [searchParams, setSearchParams] = useSearchParams(); 7 let [query, setQuery] = React.useState( 8 searchParams.get("query") 9 ); 10 11 function handleSubmit() { 12 setSearchParams({ query }); 13 } 14 15 return ( 16 <View> 17 <SearchForm onSubmit={handleSubmit}> 18 <TextInput value={query} onChangeText={setQuery} /> 19 </SearchForm> 20 </View> 21 ); 22}
